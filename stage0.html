Stage 0 Notes - Getting Started with HTML

The Web

The web is a collection of HTML documents. To browse the web, your browser uses HTTP to make requests from servers.

HTML (hypertext markup language)

HTML is made up of:

  • text content (what you see)
  • markup (what it looks like)
  • references to other documents
  • links to other pages

HTML Markup

Tags

The basic structure of tags is: <name>contents</name>

If you forget to add closing tag, open tag will apply to everything that follows it.scratchpad.io and other editors will help avoid this by autocreating closing tags for you.

A few useful examples:

  • bold tag: <b>contents</b>
  • emphasis tag (italics): <em>contents</em>
  • break tag (inline): <br> (void - does not require closing tag)
  • paragraph tag (block): <p>content</p>
Attributes

The basic structure of attributes is: <tag attr="value"> contents </tag>

A few useful examples:

  • Anchor <a> for creating links: <a href="www.reddit.com">derp</a>
  • Image tags (inline): <img src="url" alt="text">

Basic HTML Document Structure

<!DOCTYPE HTML>

<html>

<head>

<title>Title!</title>

</head>

<body>

<b>content</b>

</body>

</html>

Stage 1 Notes - Basics of HTML and CSS

HTML vs CSS

HTML defines structure of a website, CSS defines style.

Tree-like Structure of Webpages

HTML classifies page content in a branching structure, using opening and closing tags to classify elements and content.

Boxes Galore

Everything on websites is contained in boxes. These boxes are created in code through div/span tags and elements.

HTML Class Attributes

Class attributes are like labels on boxes. Class names should express purpose of code. (e.g. Title, Image, etc.)

CSS (Cascading Style Sheets)

  • Cascading - rules are applied both to elements they directly match AND all of the child elements of those elements. If a child element has multiple overlapping rules defined for it, the most specific rule is applied.
  • Style - looks/appearance to user
  • Sheets - you can have several CSS files, or sheets, all of them will be used to determine style of your page.

Search and Replace

CSS can also be thought of as a search and replace tool where you search for elements using a class or tag, and then identify what you want to do to change it (what property values to replace with different values).

Order Matters

Where you define things and the order those definitions are applied in matters. Styles are applied in the following order, and definitions further down the list overwrite previous definitions:

  1. browser default style
  2. stylesheet in a separate file
  3. stylesheet in HTML (not ideal)
  4. inline style in an element (avoid)

Selectors

Selectors determine what elements of the page a style will apply to, by selecting tag, class, etc.(e.g. all paragraphs - p {}, all items with class "description" - .description {}).

The Box Model

Nesting

Boxes have 4 nested parts. From the center outwards they are:

  1. Content
  2. Padding - space around the content, will reflect background color of box
  3. Border - comes from color property of box
  4. Margin - clear area around border, 100% transparent, "space btw boxes"

See more about nesting here

Techniques for Sizing Boxes

  • Set sizes in percentages instead of pixels.
  • Additionally, setting max-width will allow element to adjust in size as size of window is changed, but not exceed max width set.
  • Set the box-sizing attribute to border-box. This will calculate size including content, padding, and border together.

Positioning Boxes using Flex Box

To place multiple elements next to one another, change the display attribute of the parent container to value = flex, and give the items a default value less than 100%. example -

.app {

display: -webkit-flex;

display: flex;

}

Alt Attributes

Alt attributes display text in the case that images won't load and provide text descriptions of images for the visually impaired. Here is an example of how to add an alt attribute: <img src="img/app.png" alt="This is a screenshot">

Code, Test, Refine

Order of operations:

  1. Page structure - look for natural boxes to boxify design.
  2. Look for repeated styles & semantic elements - these will be defined with tags to determine headings, descriptions, etc.
  3. Write HTML.
  4. Apply styles from biggest (site-wide elements) to smallest (e.g. widgets).
  5. Fix things - test in other browsers, window sizes, mobile, etc.

DevTools

DevTools can be used to test things without changing your html & css files. Refresh the page to reset. To save changes made in DevTools, "select the source file in the 'Sources' tab, right or control+click on the file you wish to save, select 'Save As' and save the file to the directory that contains the original file. You may want to create a back-up copy of the file before making changes to it."
